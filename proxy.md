# Enterprise OAuth Token Proxy with Entra ID Certificate Authentication

## Overview

This prompt guides implementation of a **production-grade OAuth Token Proxy Service** that enables AI/LLM platforms to authenticate with Microsoft Entra ID using certificate-based authentication, while the AI platform only handles standard OAuth tokens.

### Applicable Platforms
- Cohere Connectors
- OpenAI Custom Actions/Plugins
- Anthropic Claude Tool Use
- Microsoft Copilot Studio
- LangChain/LlamaIndex Tool Integrations
- Any platform supporting OAuth 2.0 or Bearer token authentication

---

## Problem Statement

AI/LLM platforms typically support OAuth 2.0 with `client_secret` for connector authentication. However, enterprise security requirements mandate:
- Certificate-based authentication (no client secrets)
- Secrets rotation without platform reconfiguration
- Centralized credential management via Azure Key Vault
- Audit logging of all authentication events

**Solution**: A token proxy service that accepts standard OAuth requests from AI platforms and internally authenticates to Entra ID using certificates.

---

## Critical Clarification: Passwordless Architecture

### What's NOT Used (Zero Passwords)

| ❌ Not Used | Why |
|-------------|-----|
| **Usernames** | No user accounts involved — this is service-to-service (app-only) auth |
| **Passwords** | No passwords — certificate replaces the need for any password |
| **Client Secret (Entra ID)** | Migrated away from this — certificate replaces it |
| **User credentials** | App authenticates as itself, not on behalf of a user |

### What IS Used (Cryptographic, Not Passwords)

| ✅ Used | Type | Storage | Notes |
|---------|------|---------|-------|
| **Client ID** | GUID | Code/Config | Public identifier, not a secret |
| **Certificate (public)** | X.509 .cer/.crt | Entra ID App Registration | Uploaded once, used for verification |
| **Certificate (private key)** | PEM file | Azure Key Vault | Never transmitted, never leaves your infra |
| **Thumbprint** | SHA-1 hash | Code/Config | Identifier for the certificate |
| **Proxy Secret** | Random token | Key Vault + AI Platform | API key YOU generate (not a password, not from Microsoft) |

### How Certificate Auth Works (No Passwords)

```
1. Your proxy loads the PRIVATE KEY (from Key Vault)
2. Proxy creates a signed JWT assertion using the private key
3. Proxy sends assertion to Entra ID: "I am app X, here's cryptographic proof"
4. Entra ID verifies signature using the PUBLIC KEY you uploaded
5. Entra ID returns access token

No username. No password. Just cryptographic proof of identity.
```

```
┌─────────────────┐                              ┌─────────────────┐
│  Your Proxy     │   JWT signed with            │  Entra ID       │
│                 │   private key                │                 │
│  Has: Private   │ ───────────────────────────▶ │  Has: Public    │
│       Key       │                              │       Key       │
│                 │   "Signature valid,          │                 │
│                 │    here's your token"        │  Verifies       │
│                 │ ◀─────────────────────────── │  signature      │
└─────────────────┘                              └─────────────────┘

         ↑
         │
    No password
    No username
    No client secret
```

### The "Proxy Secret" Is NOT a Password

The proxy secret you generate is an **API key** for authenticating the AI platform to your proxy service. It is:

- **Generated by you** (not Microsoft, not the AI platform)
- **A random string** (e.g., `secrets.token_urlsafe(32)`)
- **Used only between** AI Platform → Your Proxy
- **NOT related to Entra ID** authentication at all

```
AI Platform → Your Proxy:  "Here's my API key (proxy secret)"
Your Proxy → Entra ID:     "Here's my cryptographic signature (certificate)"
```

Two separate authentication layers, neither uses passwords.

### Persistence in Entra ID

| Component | Location | Persistent? |
|-----------|----------|-------------|
| App Registration | Azure Portal → Entra ID | ✅ Yes |
| Public Certificate | App Registration → Certificates & Secrets | ✅ Yes (until expiry) |
| API Permissions | App Registration → API Permissions | ✅ Yes |
| Client ID | Generated by Entra ID | ✅ Yes (permanent) |

**What's NOT in Entra ID:**
- Private key (stays in your Key Vault)
- Proxy secret (you manage this)
- Any passwords (none exist)

## Implementation Requirements

### 1. Security Requirements (CRITICAL)

```yaml
Authentication:
  - Use certificate-based authentication ONLY for Entra ID
  - Generate cryptographically secure proxy secrets (min 256 bits)
  - Implement constant-time comparison for secret validation
  - Never log or expose certificates, private keys, or tokens

Secrets Management:
  - Store ALL secrets in Azure Key Vault
  - Use Managed Identity for Key Vault access (no credentials in code)
  - Implement automatic certificate rotation handling
  - Support both current and previous certificate during rotation

Transport Security:
  - Enforce TLS 1.2+ for all connections
  - Reject HTTP requests (HTTPS only)
  - Validate TLS certificates on outbound connections

Input Validation:
  - Whitelist allowed OAuth scopes
  - Validate client_id matches expected value
  - Sanitize all input parameters
  - Implement request size limits

Rate Limiting:
  - Implement per-client rate limiting
  - Add exponential backoff for failed attempts
  - Alert on anomalous request patterns

Audit Logging:
  - Log all authentication attempts (success/failure)
  - Include correlation IDs for tracing
  - Never log sensitive values (tokens, secrets, keys)
  - Send logs to centralized SIEM
```

```python
        "https://outlook.office365.com/.default",

    request_timeout_seconds: int = 30
    max_request_size_bytes: int = 10240  # 10KB
    
    # Observability
    log_level: str = "INFO"
    application_insights_connection_string: Optional[str] = None

            raise ValueError("Debug mode cannot be enabled in production")

        return v
    
    class Config:
        env_file = ".env"
        env_file_encoding = "utf-8"
        case_sensitive = False
```

#### 3.2 Certificate Authentication (`src/security/certificate_auth.py`)

```python
"""
MSAL Certificate-based authentication for Entra ID.
Handles token acquisition with automatic caching and refresh.
"""
from msal import ConfidentialClientApplication
from typing import Optional, Dict, Any, List
from functools import lru_cache
import logging

from ..config import get_settings

logger = logging.getLogger(__name__)


class CertificateAuthenticationError(Exception):
    """Raised when certificate authentication fails."""
    pass


class EntraIDCertificateAuth:
    """
    Handles certificate-based authentication to Entra ID.
    
    Security considerations:
    - Certificate and private key are loaded once at startup
    - MSAL handles token caching and refresh automatically
    - Tokens are never logged or exposed
    """
    
    def __init__(
        self,
        tenant_id: str,
        client_id: str,
        thumbprint: str,
        private_key: str,
    ):
        self._tenant_id = tenant_id
        self._client_id = client_id
        self._thumbprint = thumbprint
        self._private_key = private_key
        self._msal_app: Optional[ConfidentialClientApplication] = None
    
    @property
    def msal_app(self) -> ConfidentialClientApplication:
        """Lazy initialization of MSAL application."""
        if self._msal_app is None:
            self._msal_app = ConfidentialClientApplication(
                client_id=self._client_id,
                authority=f"https://login.microsoftonline.com/{self._tenant_id}",
                client_credential={
                    "thumbprint": self._thumbprint,
                    "private_key": self._private_key,
                },
            )
        return self._msal_app
    
    def acquire_token_for_client(
        self,
        scopes: List[str],
    ) -> Dict[str, Any]:
        """
        Acquire access token using client credentials flow with certificate.
        
        Args:
            scopes: List of scopes to request (e.g., ["https://graph.microsoft.com/.default"])
        
        Returns:
            Dict containing access_token, token_type, expires_in
        
        Raises:
            CertificateAuthenticationError: If token acquisition fails
        """
        try:
            # MSAL handles caching - will return cached token if valid
            result = self.msal_app.acquire_token_for_client(scopes=scopes)
            
            if "error" in result:
                # Log error code only, not description (may contain sensitive info)
                logger.error(
                    "Token acquisition failed",
                    extra={
                        "error_code": result.get("error"),
                        "correlation_id": result.get("correlation_id"),
                    }
                )
                raise CertificateAuthenticationError(
                    f"Authentication failed: {result.get('error')}"
                )
            
            if "access_token" not in result:
                raise CertificateAuthenticationError(
                    "No access token in response"
                )
            
            logger.info(
                "Token acquired successfully",
                extra={
                    "expires_in": result.get("expires_in"),
                    "token_source": result.get("token_source", "identity_provider"),
                }
            )
            
            return {
                "access_token": result["access_token"],
                "token_type": result.get("token_type", "Bearer"),
                "expires_in": result.get("expires_in", 3600),
            }
            
        except CertificateAuthenticationError:
            raise
        except Exception as e:
            # Never log the actual exception message (may contain secrets)
            logger.exception(
                "Unexpected error during token acquisition",
                extra={"error_type": type(e).__name__}
            )
            raise CertificateAuthenticationError(
                "Internal authentication error"
            ) from e


@lru_cache(maxsize=1)
def get_certificate_auth() -> EntraIDCertificateAuth:
    """
    Get singleton instance of certificate auth handler.
    
    In production, certificate should be loaded from Azure Key Vault.
    """
    settings = get_settings()
    
    private_key = settings.cert_private_key
    
    # If using Key Vault, load certificate from there
    if settings.key_vault_url and settings.cert_key_vault_name:
        from .key_vault import load_certificate_from_keyvault
        private_key = load_certificate_from_keyvault(
            vault_url=settings.key_vault_url,
            certificate_name=settings.cert_key_vault_name,
        )
    
    if not private_key:
        raise CertificateAuthenticationError(
            "No certificate private key configured"
        )
    
    return EntraIDCertificateAuth(
        tenant_id=settings.azure_tenant_id,
        client_id=settings.azure_client_id,
        thumbprint=settings.cert_thumbprint,
        private_key=private_key,
    )
```

#### 3.3 Secret Validation (`src/security/secret_validation.py`)

```python
"""
Secure validation of proxy secrets with timing-attack protection.
"""
import secrets
import hashlib
import logging
from typing import Optional

from ..config import get_settings

logger = logging.getLogger(__name__)


class SecretValidationError(Exception):
    """Raised when secret validation fails."""
    pass


def validate_proxy_secret(provided_secret: str) -> bool:
    """
    Validate the proxy secret using constant-time comparison.
    
    Supports both current and previous secret for rotation.
    
    Args:
        provided_secret: The secret provided by the AI platform
    
    Returns:
        True if valid, False otherwise
    
    Security:
        - Uses secrets.compare_digest for constant-time comparison
        - Supports secret rotation with previous secret fallback
        - Never logs the actual secret values
    """
    if not provided_secret:
        return False
    
    settings = get_settings()
    
    # Check current secret (constant-time comparison)
    if secrets.compare_digest(provided_secret, settings.proxy_secret):
        return True
    
    # Check previous secret if configured (for rotation)
    if settings.proxy_secret_previous:
        if secrets.compare_digest(provided_secret, settings.proxy_secret_previous):
            logger.warning(
                "Authentication using previous secret - rotation may be incomplete"
            )
            return True
    
    return False


def generate_secure_proxy_secret(length: int = 32) -> str:
    """
    Generate a cryptographically secure proxy secret.
    
    Args:
        length: Number of bytes of entropy (default 32 = 256 bits)
    
    Returns:
        URL-safe base64 encoded secret string
    """
    return secrets.token_urlsafe(length)


def hash_secret_for_logging(secret: str) -> str:
    """
    Create a safe hash of a secret for logging/debugging purposes.
    
    Returns first 8 chars of SHA256 hash - enough to identify but not reverse.
    """
    return hashlib.sha256(secret.encode()).hexdigest()[:8]
```

#### 3.4 Token Endpoint (`src/oauth/token_endpoint.py`)

```python
"""
OAuth 2.0 Token Endpoint implementation.
Proxies token requests from AI platforms to Entra ID using certificate auth.
"""
from fastapi import APIRouter, Form, HTTPException, Request, Depends
from fastapi.responses import JSONResponse
from pydantic import BaseModel
from typing import Optional, List
import logging
import uuid

from ..config import get_settings, Settings
from ..security.secret_validation import validate_proxy_secret, hash_secret_for_logging
from ..security.certificate_auth import (
    get_certificate_auth,
    CertificateAuthenticationError,
)

logger = logging.getLogger(__name__)
router = APIRouter()


class TokenResponse(BaseModel):
    """OAuth 2.0 Token Response."""
    access_token: str
    token_type: str = "Bearer"
    expires_in: int
    scope: Optional[str] = None


class TokenErrorResponse(BaseModel):
    """OAuth 2.0 Error Response."""
    error: str
    error_description: Optional[str] = None


def validate_scopes(requested_scope: Optional[str], settings: Settings) -> List[str]:
    """
    Validate and parse requested scopes against whitelist.
    
    Raises:
        HTTPException: If any scope is not allowed
    """
    if not requested_scope:
        # Default scope if none provided
        return ["https://graph.microsoft.com/.default"]
    
    scopes = requested_scope.split()
    
    for scope in scopes:
        if scope not in settings.allowed_scopes:
            logger.warning(
                "Rejected disallowed scope",
                extra={"scope": scope}
            )
            raise HTTPException(
                status_code=400,
                detail={
                    "error": "invalid_scope",
                    "error_description": f"Scope '{scope}' is not allowed"
                }
            )
    
    return scopes


@router.post(
    "/oauth/token",
    response_model=TokenResponse,
    responses={
        400: {"model": TokenErrorResponse},
        401: {"model": TokenErrorResponse},
        500: {"model": TokenErrorResponse},
    },
)
async def token_endpoint(
    request: Request,
    grant_type: str = Form(...),
    client_id: str = Form(...),
    client_secret: str = Form(...),
    scope: Optional[str] = Form(None),
    code: Optional[str] = Form(None),
    refresh_token: Optional[str] = Form(None),
    settings: Settings = Depends(get_settings),
):
    """
    OAuth 2.0 Token Endpoint.
    
    This endpoint receives standard OAuth token requests from AI platforms
    and internally authenticates to Entra ID using certificate-based auth.
    
    Supported grant types:
    - client_credentials: Service-to-service authentication
    
    Security:
    - Validates proxy secret (constant-time comparison)
    - Validates client_id matches expected value
    - Validates scopes against whitelist
    - Never logs tokens or secrets
    """
    # Generate correlation ID for request tracing
    correlation_id = str(uuid.uuid4())
    
    logger.info(
        "Token request received",
        extra={
            "correlation_id": correlation_id,
            "grant_type": grant_type,
            "client_id_hash": hash_secret_for_logging(client_id),
            "scope": scope,
        }
    )
    
    # ================================================================
    # SECURITY CHECK 1: Validate proxy secret
    # ================================================================
    if not validate_proxy_secret(client_secret):
        logger.warning(
            "Invalid proxy secret",
            extra={
                "correlation_id": correlation_id,
                "secret_hash": hash_secret_for_logging(client_secret),
            }
        )
        raise HTTPException(
            status_code=401,
            detail={
                "error": "invalid_client",
                "error_description": "Client authentication failed"
            },
            headers={"WWW-Authenticate": "Basic"},
        )
    
    # ================================================================
    # SECURITY CHECK 2: Validate client_id
    # ================================================================
    if client_id != settings.azure_client_id:
        logger.warning(
            "Client ID mismatch",
            extra={
                "correlation_id": correlation_id,
                "provided_client_id_hash": hash_secret_for_logging(client_id),
            }
        )
        raise HTTPException(
            status_code=401,
            detail={
                "error": "invalid_client",
                "error_description": "Unknown client"
            }
        )
    
    # ================================================================
    # SECURITY CHECK 3: Validate grant type
    # ================================================================
    if grant_type != "client_credentials":
        logger.warning(
            "Unsupported grant type",
            extra={
                "correlation_id": correlation_id,
                "grant_type": grant_type,
            }
        )
        raise HTTPException(
            status_code=400,
            detail={
                "error": "unsupported_grant_type",
                "error_description": f"Grant type '{grant_type}' is not supported"
            }
        )
    
    # ================================================================
    # SECURITY CHECK 4: Validate scopes
    # ================================================================
    validated_scopes = validate_scopes(scope, settings)
    
    # ================================================================
    # Acquire token using certificate authentication
    # ================================================================
    try:
        cert_auth = get_certificate_auth()
        token_result = cert_auth.acquire_token_for_client(scopes=validated_scopes)
        
        logger.info(
            "Token issued successfully",
            extra={
                "correlation_id": correlation_id,
                "expires_in": token_result["expires_in"],
            }
        )
        
        return TokenResponse(
            access_token=token_result["access_token"],
            token_type=token_result["token_type"],
            expires_in=token_result["expires_in"],
            scope=scope,
        )
        
    except CertificateAuthenticationError as e:
        logger.error(
            "Certificate authentication failed",
            extra={
                "correlation_id": correlation_id,
                "error": str(e),
            }
        )
        raise HTTPException(
            status_code=500,
            detail={
                "error": "server_error",
                "error_description": "Authentication service unavailable"
            }
        )
    except Exception as e:
        logger.exception(
            "Unexpected error in token endpoint",
            extra={
                "correlation_id": correlation_id,
                "error_type": type(e).__name__,
            }
        )
        raise HTTPException(
            status_code=500,
            detail={
                "error": "server_error",
                "error_description": "Internal server error"
            }
        )


@router.get("/health")
async def health_check():
```

```python
"""
FastAPI application entry point.
"""
from fastapi import FastAPI, Request
from fastapi.middleware.cors import CORSMiddleware
from fastapi.responses import JSONResponse
import logging
import sys

from .config import get_settings
from .oauth.token_endpoint import router as oauth_router

# Configure logging
logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    handlers=[logging.StreamHandler(sys.stdout)]
)

logger = logging.getLogger(__name__)

settings = get_settings()

app = FastAPI(
    title="Entra ID Certificate Auth Proxy",
    description="OAuth token proxy for AI/LLM platforms using certificate authentication",
    version="1.0.0",
    docs_url="/docs" if settings.environment != "production" else None,
    redoc_url=None,
)

# ================================================================
# Middleware
# ================================================================

# CORS - restrict in production
if settings.environment == "production":
    app.add_middleware(
        CORSMiddleware,
        allow_origins=[],  # No CORS in production (server-to-server only)
        allow_methods=["POST", "GET"],
        allow_headers=["Authorization", "Content-Type"],
    )


@app.middleware("http")
async def enforce_https(request: Request, call_next):
    """Enforce HTTPS in production."""
    if settings.environment == "production":
        # Check X-Forwarded-Proto for requests behind load balancer
        proto = request.headers.get("X-Forwarded-Proto", "https")
        if proto != "https":
            return JSONResponse(
                status_code=400,
                content={"error": "HTTPS required"}
            )
    return await call_next(request)


@app.middleware("http")
async def limit_request_size(request: Request, call_next):
    """Limit request body size."""
    content_length = request.headers.get("content-length")
    if content_length and int(content_length) > settings.max_request_size_bytes:
        return JSONResponse(
            status_code=413,
            content={"error": "Request too large"}
        )
    return await call_next(request)


# ================================================================
# Routes
# ================================================================

app.include_router(oauth_router)


@app.on_event("startup")
async def startup_event():
    """Validate configuration on startup."""
    logger.info(
        "Starting Entra ID Certificate Auth Proxy",
        extra={
            "environment": settings.environment,
            "tenant_id": settings.azure_tenant_id[:8] + "...",
        }
    )
    
    # Verify certificate auth is configured
    try:
        from .security.certificate_auth import get_certificate_auth
        get_certificate_auth()
        logger.info("Certificate authentication configured successfully")
    except Exception as e:
        logger.error(f"Certificate auth configuration failed: {type(e).__name__}")
        if settings.environment == "production":
            raise


@app.on_event("shutdown")
async def shutdown_event():
    """Cleanup on shutdown."""
    logger.info("Shutting down Entra ID Certificate Auth Proxy")
```

## Summary

This implementation provides:

1. **Decoupled Authentication**: AI platforms use standard OAuth, your service uses certificates
2. **Secret Isolation**: Certificates never leave your infrastructure
3. **Rotation Support**: Both proxy secret and certificate rotation without downtime
4. **Audit Trail**: Full logging of authentication events
5. **Defense in Depth**: Multiple validation layers (secret, client_id, scopes)
6. **Platform Agnostic**: Works with any AI platform supporting OAuth 2.0 or bearer tokens
